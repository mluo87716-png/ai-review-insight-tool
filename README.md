# AI Review Insight Tool

面向品牌运营、电商运营和内容运营的 **AI 消费者评论洞察工具**。

项目目标是解决品牌/电商运营中“评论很多，但不知道用户为什么买、为什么不买、该怎么吸引用户选择自家产品”的问题。它把用户评论、评分和文本反馈转化为运营人员能直接使用的消费者洞察、卖点表达、内容选题、详情页优化建议和复盘指标。

## 项目目标

让运营人员在新品上市、大促复盘或日常评论监测中，用更低成本完成“用户反馈分析 → 购买动机识别 → 卖点表达优化 → 内容/商品转化动作”的闭环，最终提升用户选择自家产品的概率。

## 一句话业务价值

这个项目不是为了“分析评论”，而是帮助运营团队从用户真实反馈里找到 **购买理由、流失阻碍和内容切入点**，再转化为商品卖点、种草内容、详情页信息和客服话术，让潜在用户更容易理解产品价值并完成选择。

## 解决的问题

| 问题 | 传统做法 | 本项目怎么解决 |
| --- | --- | --- |
| 评论太多，看不过来 | 人工逐条浏览、手动复制重点评论 | 自动识别情绪、主题和代表性痛点 |
| 不知道用户为什么选择或放弃 | 只看销量和评分，难以解释转化原因 | 从评论中提炼购买动机、阻碍因素和可放大卖点 |
| 洞察难沉淀 | 周报里只有零散用户原话 | 输出可复用的洞察总结、痛点分类和运营动作 |
| 内容选题靠感觉 | 运营凭经验想小红书/抖音选题 | 从真实用户问题生成内容选题 |
| AI 输出不敢直接用 | 通用 AI 总结容易空泛或不准确 | 增加人工复核标准和可采纳指标 |
| 结果难复盘 | 分析完就结束，没有衡量标准 | 用洞察采纳率、内容转化率、人工修正率评估价值 |

## 使用场景

- **新品上市复盘**：分析首批用户评论，判断卖点是否被用户感知，发现早期负反馈。
- **大促活动复盘**：识别价格、赠品、物流、客服等影响转化的问题。
- **商品详情页优化**：把高频疑虑前置到详情页，比如适用肤质、成分说明、价格机制。
- **内容选题生成**：从用户痛点中生成小红书、抖音、公众号和直播间 FAQ 选题。
- **客服话术沉淀**：把高频负反馈整理成客服 FAQ 和风险提醒。

## 运营作用

运营的核心不是“看数据”，而是吸引用户理解、信任并选择自家产品。本项目把评论里的真实用户声音转化为三类增长动作：

| 运营动作 | 本项目如何支持 | 对业务的意义 |
| --- | --- | --- |
| 强化购买理由 | 提炼用户已经认可的成分、效果、包装、服务等正向反馈 | 找到能被放大的核心卖点 |
| 降低选择阻碍 | 识别价格、过敏、物流、客服、使用方式等负面疑虑 | 优化详情页、FAQ 和客服话术，减少流失 |
| 生成内容入口 | 把高频问题转化为小红书、抖音、直播间和公众号选题 | 用用户真正关心的问题吸引潜在客户 |

## 运营转化链路

| 用户反馈信号 | AI 识别结果 | 运营动作 | 预期影响 |
| --- | --- | --- | --- |
| “成分写得清楚，会回购” | 用户信任成分透明度 | 详情页首屏强化成分说明，种草内容强调安全感 | 提升新用户信任和点击后的停留意愿 |
| “价格涨了，学生党不友好” | 价格成为购买阻碍 | 大促页增加组合装/小样装/价格解释卡 | 降低价格敏感用户的决策压力 |
| “敏感肌能用吗，会不会泛红” | 使用风险阻碍下单 | 增加敏感肌 FAQ、适用/不适用人群说明 | 减少下单前疑虑和售后负反馈 |
| “客服回复慢，退换货麻烦” | 服务体验影响口碑 | 沉淀客服话术和优先处理规则 | 降低负面扩散，提升服务体验 |

这条链路体现的是 AI 产品运营能力：不是只会使用 AI 工具，而是能把 AI 结果转化为影响用户选择的运营方案。

## 项目亮点

- **真实业务场景**：围绕品牌/电商运营中的评论分析、竞品监测、用户痛点提炼和内容选题展开。
- **AI 产品运营导向**：不仅输出情绪分类，还设计了“输入-分析-洞察-采纳-复盘”的完整运营工作流。
- **开箱即用**：支持无 API Key 的演示模式，HR 或面试官 clone 后可以直接跑通。
- **可扩展 AI 模式**：支持接入 Claude API，将本地规则替换为真实大模型语义分析。
- **简历友好**：项目结构包含 PRD、用户画像、指标体系、Prompt 模板和简历项目描述。
- **产品化展示**：提供本地 Demo 看板、案例复盘、竞品分析和 30 天迭代路线图，展示从工具到产品的运营思考。
- **可导出交付物**：Demo 支持上传评论文件、复制周报总结、下载 Markdown 周报和 JSON 数据。

## 快速开始

### 1. 演示模式，无需 API Key

```bash
python analyze_reviews.py --demo
```

运行后生成：

```text
output/insight_report.md
output/insight_data.json
```

### 2. AI 模式，调用 Claude API

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY="your_api_key"
python analyze_reviews.py
```

也可以指定模型：

```bash
export ANTHROPIC_MODEL="claude-3-5-sonnet-latest"
python analyze_reviews.py
```

### 3. 自定义输入和输出

```bash
python analyze_reviews.py \
  --demo \
  --input data/sample_reviews.csv \
  --output output/insight_report.md \
  --json-output output/insight_data.json
```

### 4. 打开产品 Demo

直接打开：

```text
demo/index.html
```

Demo 展示评论输入、情绪统计、痛点归类、卖点提取、内容选题建议、周报复制和数据导出。

## 面试讲法

这个项目不是算法训练，而是 AI 产品运营项目。我选择消费者评论分析这个场景，是因为运营的核心是吸引用户选择自家产品，而用户评论里正好包含“为什么买”和“为什么不买”的真实原因。我设计了一个 AI 工作流，把评论转化为购买动机、流失阻碍、可放大卖点、内容选题和运营建议，再通过人工复核和指标体系判断这些洞察是否真的能进入详情页、内容排期、客服 FAQ 和活动复盘。
