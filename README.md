# AI消费者评论洞察分析工具（AI Consumer Review Insight Analyzer）

## 项目背景

在消费品品牌实习期间，消费者评论分析是日常工作的重要部分：每周需要人工浏览大量用户评论，
手动判断情感倾向、归类痛点，再结合 SPSS 交叉分析提炼可执行的洞察，整个过程依赖人工逐条阅读，
耗时且难以规模化。

这个项目尝试用 **Claude API** 把这套"读评论 → 识别情绪/主题 → 提炼业务洞察"的流程自动化，
把原本需要几个小时人工完成的工作压缩到几分钟内完成，同时保留结构化数据供人工复核——
这正是 AI 在内容/消费者运营场景中比较实际的落地方式：不是取代判断，而是把"读完所有数据"
这一步从人力瓶颈中解放出来。

## 功能

输入一批消费者评论（CSV格式：商品名 / 评分 / 评论内容），工具自动完成：

1. **情感分类**：每条评论判断为正面/中性/负面
2. **主题归类**：自动识别评论涉及"成分透明度""价格性价比""产品效果""包装""物流""客服"等主题
3. **痛点提取**：抽取负面评论中的核心诉求
4. **洞察总结生成**：基于聚合后的统计数据，由 AI 生成一段可直接用于品牌周报的业务洞察总结

最终输出一份结构化的 `insight_report.md` 报告。

## 两种运行模式

| 模式 | 说明 | 是否需要API Key |
|---|---|---|
| **演示模式** `--demo` | 用本地规则（评分+关键词）模拟分析逻辑，方便没有API Key时也能完整跑通整个流程、快速预览效果 | 不需要 |
| **AI模式**（默认） | 每条评论真实调用 Claude API 做语义级别的情感与主题判断，并生成洞察总结 | 需要 |

这样设计是为了让任何人 clone 下来都能在 30 秒内看到完整效果，同时保留真实 AI 能力的扩展路径。

## 快速开始

```bash
# 安装依赖
pip install -r requirements.txt

# 演示模式（无需配置任何东西，直接跑）
python analyze_reviews.py --demo

# AI模式（需要先设置环境变量）
export ANTHROPIC_API_KEY="你的API Key"
python analyze_reviews.py
```

运行后会在 `output/insight_report.md` 生成报告。

## 示例输出（演示模式，基于 data/sample_reviews.csv 25条虚构样本数据）

> 本次共分析25条评论，负面占比约28%，整体情感以正面为主。用户提及最频繁的主题是"产品效果"，
> 需要重点关注。典型痛点如：用了两周脸有点泛红，客服回复很慢，问了三次才有人理我。建议品牌团队
> 优先核实并改善高频痛点对应的产品/服务环节，并在物料中强化用户认可的成分透明度卖点。

完整报告见 [`output/insight_report.md`](output/insight_report.md)。

> 说明：`data/sample_reviews.csv` 为本人编写的虚构示例数据（虚构品牌"玉肌"），
> 用于公开演示，不涉及任何实习公司的真实数据或商业机密。
