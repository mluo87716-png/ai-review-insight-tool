# JD Coverage: 岗位能力覆盖分析

## 目标

这份文档用于说明 AI Review Insight Tool 能覆盖哪些产品/运营类岗位能力，以及哪些能力还需要通过其他经历补充。

基于产品管培、商家运营、游戏运营、市场管培、AI 管培等岗位描述，本项目目标覆盖约 80% 的通用岗位能力。

## 总体覆盖结论

| 岗位方向 | 覆盖度 | 原因 |
| --- | --- | --- |
| AI 产品运营 | 85% | 有 AI 工作流、Prompt、人工复核、指标体系和 Demo |
| 产品运营 / 产品管培 | 80% | 有用户反馈分析、PRD、需求转化、路线图和跨部门交付 |
| 商家运营 / 电商运营 | 80% | 有大促复盘、价格/物流/客服问题识别、详情页优化建议 |
| 内容运营 | 85% | 有内容选题、用户痛点转化、种草话题和 FAQ 输出 |
| 用户运营 | 75% | 有用户反馈、痛点归类、复用率和满意度相关指标 |
| 市场运营 / 游戏运营 | 65% | 有市场反馈和内容策略逻辑，但缺少真实投放账户和游戏行业数据 |
| AI 管培 / 信息科技产品方向 | 65% | 有 AI 应用意识和产品化 Demo，但不覆盖深度算法、后端系统和企业级工程 |

## JD 高频能力映射

| JD 要求 | 本项目如何体现 | 可展示材料 |
| --- | --- | --- |
| 用户调研与需求分析 | 从评论中提取购买动机、流失阻碍、高频痛点和用户画像 | README、PRD、Case Study |
| 竞品与市场分析 | 对比通用 AI、数据平台和垂直运营工具，识别差异化机会 | `competitor-analysis.md` |
| 产品需求整理 | 把评论分析拆成导入、分析、复核、采纳、复盘流程 | `prd.md`、`ai-workflow.md` |
| 产品原型/功能设计 | 提供可交互 Demo，支持模板选择、评论输入、报告导出 | `docs/index.html` |
| 数据分析能力 | 输出情绪分布、主题分类、痛点提取和结构化 JSON | `output/insight_data.json` |
| 运营策略能力 | 把用户反馈转成详情页、内容选题、客服 FAQ 和大促复盘动作 | `operation-loop.md` |
| 内容运营能力 | 从真实评论生成小红书、抖音、公众号和直播间选题 | Demo 输出区、Case Study |
| 商家/电商运营能力 | 识别价格机制、物流、客服、商品详情页影响转化的问题 | README、PRD、Case Study |
| AI 工具应用 | 设计 Prompt、AI 分析流程、人工复核标准和可采纳指标 | `prompt-template.md`、`review-rubric.md` |
| 跨部门协作 | 输出可给品牌、内容、电商、客服、产品团队使用的交付物 | 周报、FAQ、JSON、运营建议 |
| 项目推进与复盘 | 设计 30 天路线图、实验计划和成功指标 | `product-roadmap.md`、`experiment-plan.md` |

## 还不能完全覆盖的能力

| 能力 | 当前不足 | 补充方式 |
| --- | --- | --- |
| 真实广告投放 | 没有真实预算、账户、投放后台数据 | 用课程/模拟投放/案例拆解补充 |
| 游戏行业理解 | 当前样例偏护肤/电商评论，不是游戏玩家数据 | 后续可增加游戏评论/玩家反馈样例 |
| 深度技术研发 | 项目是运营产品 Demo，不是算法或后端工程项目 | 不建议强行包装为技术研发能力 |
| 真实商家经营数据 | 使用公开/虚构样例，不接真实店铺后台 | 明确说明项目是流程验证和能力展示 |
| SQL/BI 深度分析 | 当前以 Python/JSON/指标框架为主 | 可后续补充简单 SQL 分析或看板截图 |

## 适配这些 JD 的项目表达重点

### 产品管培 / 产品经理助理

强调 PRD、用户反馈分析、需求转化、原型 Demo、路线图和跨部门协作。

### 商家运营 / 电商运营

强调价格机制、商品详情页、物流客服、转化阻碍、大促复盘和商家成长策略。

### 内容运营

强调用户痛点到内容选题的转化，突出小红书、抖音、直播间 FAQ 和种草内容方向。

### AI 产品运营 / AI 管培

强调 AI 工作流、Prompt、人工复核、指标体系、结构化输出和产品化落地能力。

### 游戏运营 / 市场运营

强调用户反馈分析、活动复盘、内容策略和市场趋势洞察；同时承认还需要补充游戏行业样例或投放数据。

## 80% 覆盖策略

本项目不追求覆盖所有岗位细节，而是覆盖产品/运营类岗位最通用的底层能力：

1. 看懂用户反馈。
2. 找到用户为什么选择或流失。
3. 把洞察转化为产品、内容、商家或客服动作。
4. 用指标验证动作是否有价值。
5. 用 AI 提高分析和交付效率。

这五项能力能支撑大多数产品运营、商家运营、内容运营和 AI 产品运营岗位的项目沟通与能力说明。
