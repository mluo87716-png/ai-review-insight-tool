# GitHub Pages 展示说明

## 目的

让 HR 或面试官不用下载代码，也能直接打开 Demo 页面理解项目。

## 推荐设置

1. 打开 GitHub 仓库：
   `https://github.com/mluo87716-png/ai-review-insight-tool`

2. 进入：
   `Settings` -> `Pages`

3. Source 选择：
   `Deploy from a branch`

4. Branch 选择：
   `main`

5. Folder 选择：
   `/docs`

6. 保存后等待 GitHub 生成访问链接。

## 简历链接写法

如果 GitHub Pages 开启成功，可以在简历中写：

```text
GitHub: github.com/mluo87716-png/ai-review-insight-tool
Demo: mluo87716-png.github.io/ai-review-insight-tool
```

如果没有开启 Pages，只写 GitHub 仓库链接也可以。

## 展示时怎么讲

打开 Demo 链接后，可以按这个顺序展示：

1. 先展示顶部四个指标：评论样本、高频主题、选题建议、洞察可采纳分。
2. 再展示中间工作流：评论导入、AI 分析、人工复核、运营转化。
3. 点击“加载示例”和“生成洞察”，展示情绪分布、痛点、内容选题和复盘指标。
4. 最后点击“下载周报.md”或“下载JSON”，说明这个工具不仅能看，还能把结果交付给运营周报或后续看板。
